# 电子菜单微信小程序 - 完整介绍文档

## 📱 项目概述

这是一个专为情侣/家庭设计的电子菜单微信小程序，提供完整的"点菜-备菜-做饭"闭环解决方案。通过数字化管理菜谱、食材和订单，让做饭变得更加轻松有序。

## ✨ 核心功能

### 1. 店铺管理

#### 1.1 店铺信息设置
- **店铺头像**：可自定义店铺头像图片
- **背景图**：可设置店铺背景图，展示在首页顶部
- **店铺名称**：自定义店铺名称
- **店铺简介**：添加店铺简介和公告信息

#### 1.2 多厨房支持
- 支持创建多个厨房（如：主厨房、副厨房、周末厨房等）
- 可随时切换当前使用的厨房
- 每个厨房独立管理菜谱和订单
- 支持设置默认厨房

#### 1.3 管理员设置
- 可为每个厨房设置多个管理员
- 管理员拥有以下权限：
  - 添加/编辑/删除菜谱
  - 接单和出餐操作
  - 管理厨房设置
- 普通用户只能浏览和下单

#### 1.4 订单通知
- 可开启/关闭新订单通知功能
- 新订单产生时会弹出提示

### 2. 菜谱管理

#### 2.1 添加菜谱
支持手动添加菜谱，包含以下完整信息：
- **基本信息**：
  - 菜名（必填）
  - 分类（必填）
  - 描述
  - 菜谱图片
- **制作信息**：
  - 份量（如：2-3人份）
  - 制作时长（分钟）
  - 难度（简单/中等/困难）
  - 菜谱制作链接（可跳转查看制作方法）
- **其他信息**：
  - 价格
  - 评分
  - 月销量

#### 2.2 菜谱分类
- 预设10个常用分类：
  - 田园时蔬 🥬
  - 肉肉炒菜 🥩
  - 硬核荤菜 🍖
  - 水产海鲜 🐟
  - 功夫炖汤 🍲
  - 清爽凉拌 🥗
  - 小吃速食 🍜
  - 煎炸烤卤 🍗
  - 炸锅美食 🍤
  - 再来亿碗 🍚
- 支持按分类筛选浏览菜谱

#### 2.3 必点菜设置
- 可标记某些菜谱为"必点菜"
- 必点菜在列表中优先显示
- 方便快速找到推荐菜品

#### 2.4 搜索功能
- 支持按菜名搜索
- 支持按描述搜索
- 支持按分类搜索
- 实时搜索，即时显示结果

#### 2.5 批量导出
- 支持导出所有菜谱数据为JSON格式
- 数据包含完整的菜谱信息
- 可用于数据备份

#### 2.6 删除菜谱
- 支持删除不需要的菜谱
- 删除前会弹出确认提示
- 删除操作不可恢复

### 3. 点菜与订单

#### 3.1 浏览菜单
- 以卡片形式展示所有菜谱
- 显示菜谱图片、名称、评分、月销量等信息
- 支持分类筛选
- 支持搜索功能
- 必点菜优先显示

#### 3.2 菜谱详情
- 查看完整的菜谱信息
- 包括：
  - 大图展示
  - 菜名、分类、描述
  - 份量、制作时长、难度
  - 制作链接（可复制）
  - 评分和销量信息

#### 3.3 快速下单
- 从菜谱详情页直接下单
- 支持加入购物车
- 购物车显示商品数量
- 一键下单功能

#### 3.4 订单管理
- **查看所有订单**：列表形式展示所有订单
- **按状态筛选**：
  - 全部订单
  - 待处理订单
  - 制作中订单
  - 已完成订单
- **接单功能**（管理员）：
  - 管理员可以将待处理订单标记为"制作中"
  - 接单后订单状态更新
- **出餐功能**（管理员）：
  - 管理员可以将制作中订单标记为"已完成"
  - 出餐前会弹出确认提示

#### 3.5 订单详情
- 查看订单完整信息
- 包括：
  - 订单编号
  - 下单时间、更新时间
  - 订单状态
  - 订单商品列表（图片、名称、数量、价格）
  - 订单总价
  - 订单备注（如有）

### 4. 数据管理

#### 4.1 数据导出
- 导出所有数据为JSON格式
- 包含：
  - 店铺信息
  - 所有菜谱
  - 所有订单
  - 分类信息
- 数据复制到剪贴板，可粘贴保存

#### 4.2 数据导入
- 从剪贴板导入JSON数据
- 支持恢复之前导出的数据
- 导入后自动刷新所有页面

#### 4.3 数据统计
- 显示当前厨房的菜谱数量
- 显示订单总数
- 显示待处理订单数
- 显示已完成订单数

#### 4.4 清空数据
- 支持清空所有数据
- 清空前会弹出确认提示
- 清空后重新初始化默认数据

## 🚀 快速开始

### 1. 环境准备
- 安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
- 注册微信小程序账号（测试可使用测试号）

### 2. 导入项目
1. 打开微信开发者工具
2. 选择"小程序"项目类型
3. 点击"+"创建新项目
4. 填写项目信息：
   - **项目名称**：电子菜单
   - **目录**：选择本项目的 `CookingApp2` 文件夹
   - **AppID**：填写你的小程序 AppID（测试可使用测试号）

### 3. 准备图片资源
在 `images/tabbar/` 目录下准备以下图标：
- `kitchen.png` / `kitchen-active.png` - 厨房图标
- `order.png` / `order-active.png` - 订单图标
- `profile.png` / `profile-active.png` - 个人中心图标

> **提示**：如果暂时没有图片，可以先使用占位图，小程序会正常显示。

### 4. 运行项目
1. 在微信开发者工具中点击"编译"
2. 在模拟器中预览小程序
3. 使用真机调试功能在手机上测试

## 📖 使用指南

### 首次使用

1. **设置店铺信息**
   - 进入"我的" → "店铺设置"
   - 设置店铺头像、背景、名称和简介
   - 保存设置

2. **创建厨房**
   - 在店铺设置中点击"管理厨房"
   - 添加新厨房（如：主厨房、周末厨房）
   - 设置默认厨房

3. **添加管理员**
   - 在厨房列表中点击"管理员"
   - 添加管理员昵称
   - 管理员可以管理菜谱和订单

4. **添加菜谱**
   - 在厨房页面点击"添加菜谱"
   - 填写菜谱信息（菜名、分类必填）
   - 上传菜谱图片
   - 保存菜谱

5. **开始点菜**
   - 在厨房页面浏览菜谱
   - 点击菜谱查看详情
   - 点击"+"加入购物车或直接下单

### 日常使用

1. **点菜流程**
   - 浏览菜谱 → 加入购物车 → 点击"下单"
   - 或直接点击菜谱详情页的"快速下单"

2. **订单管理**
   - 管理员在"订单"页面查看所有订单
   - 点击"接单"开始制作
   - 制作完成后点击"出餐"完成订单

3. **菜谱管理**
   - 管理员切换到"编辑"模式
   - 可以编辑或删除菜谱
   - 可以添加新菜谱

4. **数据备份**
   - 在"我的"页面点击"导出数据"
   - 数据会复制到剪贴板
   - 粘贴保存到安全位置

5. **数据恢复**
   - 在"我的"页面点击"导入数据"
   - 将之前导出的JSON数据复制到剪贴板
   - 点击确定导入

## 🔧 技术特点

### 数据存储
- 使用微信小程序本地存储（wx.setStorageSync）
- 数据持久化保存
- 支持数据导出和导入

### 权限管理
- 基于管理员列表的权限控制
- 普通用户只能浏览和下单
- 管理员可以管理菜谱和订单

### 多厨房支持
- 每个厨房独立管理
- 菜谱和订单按厨房隔离
- 支持快速切换厨房

### 用户体验
- 流畅的页面切换
- 清晰的视觉反馈
- 友好的错误提示
- 符合微信小程序设计规范

## 📝 注意事项

1. **图片资源**
   - 需要自行准备TabBar图标
   - 默认图片路径已配置，但需要添加实际图片文件
   - 建议使用压缩后的图片以提升性能

2. **数据安全**
   - 数据存储在本地，建议定期导出备份
   - 清空数据操作不可恢复，请谨慎操作

3. **权限设置**
   - 只有管理员才能管理菜谱和订单
   - 普通用户需要管理员添加后才能获得权限

4. **兼容性**
   - 支持微信小程序基础库 2.19.4 及以上版本
   - 建议在真机上测试所有功能
